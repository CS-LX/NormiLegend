# 策略节点编辑器 - 使用指南

## 概述

策略节点编辑器是一个 Blender/Unity 风格的可视化节点图工具，用于为关卡中的物件（触发器、机关、敌人等）配置行为逻辑。通过拖拽节点和连线，可以直观地设计"当什么条件满足时，执行什么动作"的策略。

---

## 打开与关闭

| 操作 | 方式 |
|------|------|
| 打开 | 在关卡编辑器中，选中物件 → 属性面板中点击「打开节点编辑器」 |
| 关闭 | 按 **ESC** 键，返回关卡编辑器 |

---

## 基本操作

| 操作 | 方式 |
|------|------|
| 平移画布 | 鼠标中键拖拽 / 在空白区域左键拖拽 |
| 缩放画布 | 鼠标滚轮（以鼠标位置为中心） |
| 添加节点 | 在空白区域**右键**，弹出分类菜单 |
| 选中节点 | 左键单击节点 |
| 移动节点 | 左键拖拽节点标题区域 |
| 删除节点 | 选中后按 **Delete** 键（事件入口节点不可删除） |
| 连接端口 | 从一个端口拖拽到另一个端口 |

## 端口类型与连线规则

节点之间通过端口（Port）连线传递数据或控制流。

| 端口类型 | 外观 | 颜色 | 用途 |
|---------|------|------|------|
| **flow** (执行流) | 三角形 ▶ | 白色 | 控制执行顺序 |
| **number** (数值) | 圆形 ● | 绿色 | 传递浮点数 |
| **boolean** (布尔) | 圆形 ● | 橙色 | 传递 真/假 |

**连线规则**：
- 只能连接相同类型的端口（flow↔flow, number↔number, boolean↔boolean）
- 输出端口 → 输入端口（方向不可反）
- 一个输出可连接多个输入

## Inspector 属性面板

选中节点后，右侧面板显示该节点的可编辑属性：

- **数值类型**：点击左侧「−」减小，点击右侧「+」增大
- **选择类型**：点击值区域循环切换选项，或点击右侧「▸」
- **文本类型**：显示当前值（暂为只读显示）
- **布尔类型**：点击切换 开/关

Inspector 支持滚轮滚动查看更多属性。

## 节点分类详解

### 入口节点

| 节点 | 图标 | 说明 |
|------|------|------|
| **事件入口** | ▶ | 逻辑起点，每个策略图自动包含一个，不可删除 |

---

### 数据节点

| 节点 | 图标 | 说明 | 属性 |
|------|------|------|------|
| **常量** | # | 输出一个固定数值 | 数值 (-9999~9999) |
| **读取变量** | $ | 读取运行时变量 | 变量名 (见下表) |

**可读取的运行时变量**：

| 变量 | 含义 |
|------|------|
| playerX / playerY | 玩家坐标 |
| playerHP / playerMP | 玩家血量/魔力 |
| playerSpeedX / playerSpeedY | 玩家速度 |
| playerOnGround | 玩家是否着地 |
| triggerX / triggerY | 触发器坐标 |
| enemyCount | 场景敌人数 |
| time | 已用时间 |
| hitCount / killCount | 被击/击杀次数 |
| custom1 / custom2 / custom3 | 自定义变量 |

---

### 条件节点

| 节点 | 图标 | 说明 | 属性 |
|------|------|------|------|
| **比较** | ? | 比较两个数值，输出布尔值 | 运算符: >, <, ==, !=, >=, <= |
| **逻辑** | & | 逻辑运算，输出布尔值 | 运算符: 且(AND), 或(OR), 非(NOT) |

---

### 运算节点

| 节点 | 图标 | 说明 | 属性 |
|------|------|------|------|
| **数学** | + | 对两个数值做运算 | 运算符: +, -, x, ÷, %, min, max, abs, clamp |

---

### 流程控制节点

| 节点 | 图标 | 说明 | 属性 |
|------|------|------|------|
| **条件分支** | ◇ | if/then/else 分支 | 输入条件(boolean)，输出 True/False 两条流 |
| **顺序执行** | ↓ | 按顺序依次执行子节点 | 动态 children |
| **随机选择** | ? | 加权随机选择一个子节点执行 | 动态 children |
| **延迟** | ⏱ | 等待 N 秒后继续执行 | 延迟秒数 (0~60) |
| **重复** | ↺ | 重复执行 N 次 | 重复次数 (1~100) |

---

### 动作节点

| 节点 | 图标 | 说明 | 关键属性 |
|------|------|------|---------|
| **生成实体** | ★ | 生成敌人/道具/特效 | 实体类型、数量、坐标(X/Y输入端口) |
| **移动对象** | → | 平移/旋转场景对象 | 目标ID、持续时间、缓动方式 |
| **设置变量** | = | 修改运行时参数 | 变量名、方式(设为/增加/乘以)、新值 |
| **播放效果** | ♪ | 音效/震屏/粒子/提示 | 效果类型、参数、强度 |
| **门/开关** | ⊞ | 控制门/阻挡物 | 动作(打开/关闭/切换/移除)、目标ID |
| **传送** | ⊕ | 传送玩家到指定位置 | 目标坐标(X/Y输入端口)、相对/绝对 |
| **伤害/治疗** | ♥ | 对玩家造成伤害或治疗 | 数值、是否治疗模式 |
| **胜负判定** | 🏆 | 触发过关/失败/重开 | 结果类型 |

**可生成的实体**：近战敌人、远程敌人、飞行敌人、Boss、生命道具、魔力道具、金币、钥匙、爆炸特效、冰霜特效、移动平台、投射物

**播放效果类型**：播放音效、相机抖动、全屏闪光、浮动文字、粒子效果、慢动作

## 典型用法示例

### 示例1：玩家进入区域后生成3个敌人

```
[事件入口] ──flow──► [生成实体]
                      实体类型: 近战敌人
                      数量: 3
                      X ◄── [常量: 500]
                      Y ◄── [常量: 200]
```

### 示例2：血量低于30时播放警告音效

```
[事件入口] ──flow──► [条件分支]
                      条件 ◄── [比较: <]
                                 A ◄── [读取变量: playerHP]
                                 B ◄── [常量: 30]
                      True ──► [播放效果]
                                效果类型: 播放音效
                                参数: "warning"
```

### 示例3：击杀所有敌人后开门

```
[事件入口] ──flow──► [条件分支]
                      条件 ◄── [比较: ==]
                                 A ◄── [读取变量: enemyCount]
                                 B ◄── [常量: 0]
                      True ──► [门/开关]
                                动作: 打开
                                目标ID: "gate_01"
```

### 示例4：延迟2秒后连续生成3波敌人

```
[事件入口] ──flow──► [延迟: 2秒] ──► [重复: 3次]
                                       循环体 ──► [生成实体]
                                                    实体类型: 远程敌人
                                                    数量: 2
```

## 快捷键总览

| 快捷键 | 功能 |
|--------|------|
| ESC | 关闭节点编辑器，返回关卡编辑器 |
| 右键 | 在画布空白处弹出添加节点菜单 |
| Delete | 删除选中的节点 |
| 鼠标滚轮 | 缩放画布 |
| 中键拖拽 | 平移画布 |
| 左键空白拖拽 | 平移画布 |
| 左键节点拖拽 | 移动节点 |
| 左键端口拖拽 | 连线 |

---

## 注意事项

1. **事件入口节点**是每个策略图的起点，不可删除
2. 连线只能在**相同类型**的端口之间建立
3. 执行流（flow）决定"做什么"，数据流（number/boolean）决定"用什么值"
4. 节点属性在 Inspector 面板中编辑，选中节点后右侧自动显示
5. 策略图数据会自动保存到物件属性中，关闭编辑器即生效
